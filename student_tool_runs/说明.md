## 启动前检查

先启动后端：

```powershell
.\.venv\Scripts\python.exe -m uvicorn medagent.api.app:app --reload --port 8001
```

检查工具状态：

```powershell
curl.exe http://127.0.0.1:8001/tools/status
```

需要记录 RDKit、Chemprop、GNINA、Vina、DiffDock、AiZynthFinder 哪些可用。Docker 没启动时，外部 docking 或合成路线工具可能不可用，这是正常现象，也需要写进结果。

## BRAF 对接与候选细筛复现

复现 5 个 BRAF 候选分子的 docking/外部细筛结果，观察外部工具是否真的被调用，以及结果是否接近项目已有基线。

使用数据：

- `student_tool_runs/student_A_docking_assessment/docking_panel.csv`
- `student_tool_runs/common/braf_external_tool_baseline.csv`
- `student_tool_runs/common/braf_binding_site.json`
- `student_tool_runs/data/3OG7_BRAF_receptor.pdb`
- `student_tool_runs/data/ligands/*.sdf`
- `student_tool_runs/data/poses/*.sdf`

建议调用：

- `/tools/status`
- `/tools/docking/run`
- `/projects/PROJ-8A2756E8CB/candidate-assessment/run`

已准备好的请求文件：

- 单分子 docking：`student_tool_runs/student_A_docking_assessment/single_docking_requests.jsonl`
- fast 粗筛：`student_tool_runs/student_A_docking_assessment/candidate_assessment_fast_payload.json`
- external top5 细筛：`student_tool_runs/student_A_docking_assessment/candidate_assessment_external_top5_payload.json`

交回内容：

- 填好的 `student_tool_runs/student_A_docking_assessment/result_template.csv`
- 简短说明：哪些工具实际可用、每个分子的 vina_score/cnn_score 是否接近基线、fast 和 external 的结果有什么区别。

重点观察：

- `tool_name` 和 `adapter_mode` 是外部工具还是 surrogate。
- Docker 启动前后 `/tools/status` 是否变化。
- `MOL-AB0A4C6BC4` 的 docking 分数明显异常，可作为反例讨论。

## Top1 类似物性质、ADMET 与合成可行性筛选

任务目标：围绕当前 Top1 分子做一个小型类似物面板，用 RDKit/ADMET/合成路线结果筛选 3 个值得继续优化的分子。

使用数据：

- `student_tool_runs/student_B_property_synthesis/top1_analog_panel.csv`
- `student_tool_runs/student_B_property_synthesis/top1_analog_rdkit_baseline.csv`
- `student_tool_runs/common/braf_top10_agent_baseline.csv`
- `student_tool_runs/data/retrosynthesis/*/aizynthfinder_routes.json`

建议调用：

- `/tools/rdkit/validate`
- `/tools/admet/predict`
- 可选：`/projects/PROJ-8A2756E8CB/candidate-assessment/run`

已准备好的请求文件：

- RDKit 验证：`student_tool_runs/student_B_property_synthesis/rdkit_validate_requests.jsonl`
- ADMET 预测：`student_tool_runs/student_B_property_synthesis/admet_predict_payload.json`
- 现有项目候选对照评估：`student_tool_runs/student_B_property_synthesis/candidate_assessment_existing_controls_payload.json`

交回内容：

- 填好的 `student_tool_runs/student_B_property_synthesis/result_template.csv`
- 选出 3 个分子进入下一轮优化，并用 3-5 句话说明理由。

重点观察：

- `RotB`、`QED`、`LogP`、`TPSA` 对候选选择的影响。
- Top1 的小改造是否比复杂 Top2 更适合作为下一轮优化起点。
- ADMET 或合成路线不理想时，是否应该降低排序优先级。

## 注意事项

- 不要默认跑 full 模式；full 只适合最后小规模验收。
- 如果外部工具不可用，先记录状态，不要强行改代码。
- 结果可以和 `common/` 里的基线表对比，差异较大时优先检查 Docker、工具路径、输入 ligand 文件和 grid 参数。
